主要的文件结构（重要，简化）
```
/prompt iteration/
├── README.md （引导、入口）
├── 角色模板（role-template）/
├── 现实模板（reality-tracker）/
├── 角色-现实对比分析报告（role-vs-reality）/
├── 任务包/
│   ├── 任务1：任务包1.md
│   ├── 任务2：任务包2.md
│   └── ...
├──Copoilt_agreement.md

说明：
1、Copoilt_agreement是具有任务有限调度、模块化设计和具备高度个性化的学习prompt。用于本人通过AI来进行学习

2、角色模板（role-template）/
该文件夹下存放迭代后的Copoilt_agreement版本，会对一些模块功能和个人目标进行改进和修正

3、现实模板（reality-tracker）/
该文件夹下存放的是通过让AI了解我目前的各种情况和问题，生成的阶段性记录模板。用于和角色模板中进行对比，帮助我改善自身缺陷和学习方式

4、角色-现实对比分析报告（role-vs-reality）/
该文件会存放通过上一次AI对话交流中生成的现实模板，将它和角色模板进行对比和分析，生成比较报告，帮助我识别问题和获取改进建议

5、任务包/
存放生成和待完成的任务，按照一定的需求和内容生成的打包内容。用于下一次对话中，生成内容和帮助作者完成任务
```

其他说明：
```
角色模板：3个月更新一次
现实模板：每月/每周更新一次
角色-现实对比分析报告：每季度生成一次
```
